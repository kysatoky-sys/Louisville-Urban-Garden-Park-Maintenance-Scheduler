# Louisville Urban Garden & Park Maintenance Scheduler

An ETL pipeline that fetches 16-day hourly weather and soil data from the Open-Meteo API for eight Louisville, KY parks, aggregates it to daily values, and produces a maintenance schedule that recommends, defers, or skips each task — mowing, irrigation, fertilizing, planting, pruning — based on weather constraint rules stored in the database.

---

## How It Works

The pipeline runs in two scripts executed in order.

| Step | Script | Reads | Writes |
| --- | --- | --- | --- |
| 1 | `extract_transform.py` | `data/seed_parks.csv` + Open-Meteo API | `data/weather_data.csv` |
| 2 | `load.py` | `data/weather_data.csv` + `data/seed_parks.csv` + `data/seed_maintenance_tasks.csv` | Supabase PostgreSQL |

**`extract_transform.py`** reads park coordinates from `seed_parks.csv`, calls the Open-Meteo hourly forecast API once per park, aggregates hourly readings into daily means and totals, validates the data, and writes `weather_data.csv`. It assigns each park a stable integer ID and stamps it onto every row — this is the single place park IDs are created for the whole pipeline. No database connection is needed.

**`load.py`** reads `weather_data.csv` first, derives park IDs from it by joining on park name (not row position), reads the two seed CSVs, evaluates each task's weather constraint rules against the daily weather data to compute the maintenance schedule, creates the database schema, and inserts all four tables.

## Prerequisites

- Python 3.11 or higher
- A [Supabase](https://supabase.com) project (free tier is sufficient)
- Your Supabase database password and project reference ID

No API key is required. Open-Meteo is a free, open-source weather API.

## Setup Instructions

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/louisville-park-scheduler.git
cd louisville-park-scheduler
```

### 2. Create and activate a virtual environment

```bash
# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate

# Windows
python -m venv .venv
.venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your Supabase credentials:

```
DB_PASSWORD=your_supabase_database_password
DB_REF=your_supabase_project_ref
```

Your project reference ID is the alphanumeric string in your Supabase dashboard URL:
`https://supabase.com/dashboard/project/<DB_REF>`

Alternatively, paste a full connection string (this overrides `DB_PASSWORD` and `DB_REF`):

```
SUPABASE_DB_URL=postgresql+psycopg2://postgres:<password>@db.<ref>.supabase.co:5432/postgres
```

## Re-running the Pipeline

Both scripts are fully idempotent and require no manual edits between runs. Re-running refreshes the 16-day forecast window and updates the schedule automatically.

`load.py` uses `ON CONFLICT ... DO UPDATE` for `weather_forecast` and `maintenance_schedule`, so duplicate rows are never created on re-run.

By default `RESET_TABLES=true`, which drops and recreates all tables on each `load.py` run for a clean reload. To upsert into existing tables without dropping them:

```bash
RESET_TABLES=false python load.py
```

## Verifying the Data in Supabase

Open the Supabase SQL Editor and run:

```sql
-- Row counts for all four tables
SELECT 'park'                  AS tbl, COUNT(*) AS rows FROM public.park
UNION ALL
SELECT 'maintenance_task',     COUNT(*) FROM public.maintenance_task
UNION ALL
SELECT 'weather_forecast',     COUNT(*) FROM public.weather_forecast
UNION ALL
SELECT 'maintenance_schedule', COUNT(*) FROM public.maintenance_schedule;
```

```sql
-- Today's maintenance schedule across all parks
SELECT
    p.name            AS park,
    mt.task_name      AS task,
    ms.schedule_date,
    ms.recommendation,
    ms.reason
FROM  public.maintenance_schedule ms
JOIN  public.park             p  ON p.park_id  = ms.park_id
JOIN  public.maintenance_task mt ON mt.task_id = ms.task_id
WHERE ms.schedule_date = CURRENT_DATE
ORDER BY p.name, mt.task_name;
```

```sql
-- Weather data for one park
SELECT
    p.name,
    wf.forecast_date,
    wf.soil_temperature_0cm,
    wf.soil_moisture_0_to_1cm,
    wf.evapotranspiration,
    wf.precipitation
FROM  public.weather_forecast wf
JOIN  public.park p ON p.park_id = wf.park_id
WHERE p.name = 'Cherokee Park'
ORDER BY wf.forecast_date;
```

## Validation Checks

Eight validation checks run across the two scripts. Full documentation is in `validation_framework.md`.

| # | Check | Script | Action on failure |
| --- | --- | --- | --- |
| 1 | API response validation | `extract_transform.py` | Park skipped; error logged; other parks continue |
| 2 | Null value check | `extract_transform.py` | Filled with column median; warning logged |
| 3 | Range validation | `extract_transform.py` | Clipped to valid bounds; warning logged |
| 4 | Duplicate detection | `extract_transform.py` | Duplicates dropped; warning logged |
| 5 | Output row count check | `extract_transform.py` | Pipeline halts; no CSV written |
| 6 | JSON schema validation | `load.py` | Row kept with empty rule set; error logged |
| 7 | Referential integrity check | `load.py` | Pipeline halts before any child rows inserted |
| 8 | Post-load row count check | `load.py` | Pipeline halts with expected vs actual counts |

## Troubleshooting

**`FileNotFoundError: data/weather_data.csv not found`**
Run `extract_transform.py` before `load.py`.

**`RuntimeError: Set SUPABASE_DB_URL, or set both DB_PASSWORD and DB_REF`**
Your `.env` file is missing or the variables are not set. Confirm `.env` exists in the project root and the values are filled in.

**`psycopg2.OperationalError: could not connect to server`**
Supabase free-tier projects pause after inactivity. Open your Supabase dashboard to wake the project, then retry.

**`WARNING — No weather data found for 1 park(s): ['Park Name']`**
The Open-Meteo API fetch failed for that park after 5 retries. The park is still inserted into the `park` table but will have no weather or schedule rows. Check `logs/etl.log` for the specific error and retry.
